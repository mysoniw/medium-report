# Medium 아티클 분석 리포트 생성

사용자가 블로그 URL을 제공하면 분석 리포트를 생성한다.

## 트리거
- `report`, `리포트`, `보고서`, `분석해줘`, `정리해줘` + URL
- `/report <URL>`

## 프로세스

### Step 1: 아티클 수집
1. CMUX 브라우저로 URL 열기
   ```bash
   cmux browser open-split <URL>
   ```
2. 페이지 로드 대기 후 본문 전체 추출
   ```bash
   cmux browser --surface <ID> wait --load-state complete
   cmux browser --surface <ID> eval "document.querySelector('article')?.innerText"
   ```
3. 제목, 저자, 날짜, 읽기 시간 등 메타데이터 수집

### Step 2: 폴더 생성
- 프로젝트 루트: `/Users/minholee/projects/medium/`
- 폴더명: URL slug 기반 (예: `boris-cherny-claude-code-tips-skill/`)
- 리포트 파일: `{slug}/report.md`

### Step 3: 리포트 작성

아래 구조를 **반드시** 따른다:

```markdown
# 한국어 제목

> 원문: [영문 제목](URL) — 저자명, 날짜

---

## 1. 요약
- 3-5문장으로 기사의 핵심 내용과 저자의 주장 요약
- 핵심 수치/데이터가 있으면 포함

## 2. 상세 내용
- 기사의 논리 구조를 따라 하위 섹션(2.1, 2.2, ...)으로 분해
- 코드 예시, 비교표, 다이어그램 활용
- 저자의 실전 경험/데이터가 있으면 별도 표로 정리

## 3. 핵심 인사이트 정리
- 번호 매긴 리스트 (5-7개)
- 각 인사이트는 "결론 + 근거" 구조

## 4. 원문 영어 표현 해설

4개 카테고리로 분류:

### 핵심 개념어
| 원문 | 직역 | 저자가 의도한 뉘앙스 |
원문 표현 → 직역 → 저자가 그 단어를 선택한 이유, 문화적 맥락

### 비유·수사 표현
| 원문 | 직역 | 문화적 맥락 |
비유의 원천, 한국어 유사 표현 대응 (예: blast radius → 폭발 반경 → DevOps 차용)

### 업계 전문 용어
| 원문 | 의미 | 맥락 |
해당 용어가 어떤 분야에서 왜 쓰이는지

### 저자 어투 분석
저자의 글쓰기 스타일, 수사 패턴, 이전 보고서 저자와의 비교

---

*카테고리: {카테고리}*
*태그: `#tag1` `#tag2` ...*
*키워드: 쉼표로 구분된 키워드 목록*
*관련 문서: 기존 보고서 중 연관된 것 링크*
*Generated: {날짜}*
```

### Step 4: index.md + index.html 업데이트

#### 4-1. index.md (3개 섹션 모두 업데이트)

`/Users/minholee/projects/medium/index.md`:

1. **문서 목록 테이블**: 새 행 추가 (번호, 제목, 카테고리, 태그, 저자, 날짜, 경로)
2. **카테고리 섹션**: 해당 카테고리에 항목 추가 (없으면 카테고리 생성)
3. **태그 인덱스**: 새 태그 행 추가 + 기존 태그에 문서 링크 추가

#### 4-2. index.html (articles 배열에 추가)

`/Users/minholee/projects/medium/index.html`의 `const articles = [...]` 배열에 새 항목 추가:

```js
{
  id: {다음 번호},
  title: "한국어 제목",
  description: "2-3문장 요약",
  category: "카테고리",
  tags: ["tag1", "tag2", ...],
  author: "원문 저자",
  date: "YYYY-MM-DD",
  path: "{slug}/report.md",
  originalUrl: "원문 URL",
  keywords: ["검색용", "키워드", "목록", ...]
}
```

**주의**: id는 기존 마지막 id + 1, keywords는 15-20개로 한국어+영어 혼합

### Step 5: 완료 보고

리포트 생성 후 사용자에게 보고:
- 리포트 경로
- 추출된 카테고리, 태그
- 관련 기존 문서 링크
- 지식 그래프 연결점 (기존 문서와 공유하는 태그/키워드)

## 메타데이터 규칙 (지식 그래프용)

- **카테고리**: 기존 카테고리 우선 재사용, 없으면 새로 생성
  - 현재: `AI/에이전트 엔지니어링`, `개발 도구/생산성`, `학습/생산성`
- **태그**: 5-8개, `#kebab-case`, 기존 태그와 최대한 연결
- **키워드**: 검색용 한국어+영어 혼합, 쉼표 구분
- **관련 문서**: 태그가 2개 이상 겹치는 기존 문서 자동 링크

## 품질 기준

- 단순 번역이 아닌 **분석과 재구성**
- 저자의 주장과 근거를 분리하여 서술
- 실전 데이터/수치가 있으면 반드시 포함
- 영어 뉘앙스 해설은 단순 사전 풀이가 아닌, 저자가 그 단어를 선택한 이유와 문화적 배경까지 설명
