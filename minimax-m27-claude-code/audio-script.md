# MiniMax M2.7을 Claude Code에 연결했더니 하룻밤 만에 비용 낭비가 멈췄다

안녕하세요. 오늘은 AI 코딩 도구의 비용을 획기적으로 줄일 수 있는 방법에 대해 이야기해 드릴게요. Medium에 올라온 개발자 Joe Njenga의 경험을 바탕으로, MiniMax M2.7이라는 모델을 Claude Code에 연결하면 어떤 일이 벌어지는지 차분히 살펴볼게요.

많은 개발자들이 Claude Code를 사용하면서 비용 문제를 느끼고 있어요. Claude Opus나 Sonnet 같은 고급 모델을 매일 쓰다 보면 토큰 비용이 생각보다 빠르게 쌓이거든요. Joe도 마찬가지였어요. 그는 이미 이전에 MiniMax M2.5나 GLM 5 같은 대안 모델들을 Claude Code에 붙여본 경험이 있었는데, M2.7이 출시됐을 때 처음에는 그냥 지나쳤다고 해요. 그런데 벤치마크 수치를 보고 나서 직접 테스트해보기로 결심했어요.

MiniMax M2.7은 중국 AI 스타트업 MiniMax가 2026년 3월 18일에 출시한 오픈소스 모델이에요. 이 모델의 가장 큰 특징은 아키텍처에 있어요. 전체 파라미터는 230B에 달하지만, 실제로 쿼리 하나를 처리할 때는 약 10B만 활성화되는 Sparse MoE, 즉 희소 전문가 혼합 구조를 사용해요. 덕분에 대형 모델 수준의 성능을 훨씬 낮은 계산 비용으로 제공할 수 있어요.

컨텍스트 윈도우는 204,800 토큰으로 상당히 넓고, MIT 라이선스로 오픈소스로 공개되어 있어요. 가격은 입력 토큰 기준으로 백만 토큰당 0.30달러, 출력은 1.20달러예요. Claude Sonnet과 비교하면 약 8% 수준의 비용이에요.

벤치마크 성능도 주목할 만해요. 소프트웨어 엔지니어링 능력을 평가하는 SWE-Pro에서 56.22%, 터미널 기반 코딩 평가인 Terminal Bench 2에서 57.0%를 기록했어요. 복잡한 스킬 40개를 얼마나 일관성 있게 수행하는지 측정하는 스킬 준수율은 97%에 달했고요.

또 한 가지 중요한 기능이 있어요. 바로 인터리브드 씽킹이에요. 도구를 호출하는 사이사이에 추론 과정을 삽입할 수 있는 기능인데, 단순히 명령을 순서대로 실행하는 게 아니라 각 단계마다 상황을 판단하면서 유연하게 대응할 수 있어요. 에이전트 환경에서 특히 중요한 능력이에요.

그럼 이 모델을 Claude Code에 어떻게 연결하느냐고요? 놀랍도록 간단해요. Claude Code는 Anthropic 호환 API를 지원하는 어떤 모델과도 연동이 가능한 구조로 설계되어 있어요. 홈 디렉토리의 settings.local.json 파일에서 모델과 API 베이스 URL 두 가지만 바꿔주면 돼요. 설정 변경 후에는 기존에 만들어둔 에이전트, 슬래시 커맨드, MCP 도구, 훅 등이 그대로 동작해요. 워크플로우를 전혀 바꾸지 않고 모델만 교체되는 거예요.

Joe는 여기서 한 걸음 더 나아가 Ollama를 통한 로컬 배포 방식도 활용했어요. 이렇게 하면 API 비용 자체가 없어지고, 코드가 외부 서버로 전송되지 않아 프라이버시도 보장돼요. 인터넷 없이도 작동하고요. 민감한 코드베이스를 다루는 기업 환경이라면 특히 매력적인 옵션이에요.

실제 테스트 결과를 볼게요. Joe는 단순한 CRUD 앱이 아니라 진짜 복잡한 코딩 시나리오에서 테스트했어요. 전체 프로젝트 단위 코드 생성, 프로덕션 디버깅, 코드 보안 분석, 복잡한 시스템 이해와 수정 같은 작업들이요. 같은 작업 기준으로 비용을 비교해보니 MiniMax M2.7은 0.09달러, Claude Sonnet은 1.05달러가 나왔어요. 약 열 배 이상 차이가 나는 거예요.

물론 MiniMax M2.7이 모든 상황에서 Claude를 완전히 대체할 수 있는 건 아니에요. 복잡한 아키텍처 결정이 필요한 상황에서는 여전히 Claude가 우세하다고 해요. 주의해야 할 점도 두 가지가 있어요. 첫째로 모델 버전 고정을 지원하지 않아서, 공지 없이 모델이 업데이트될 수 있어요. 오늘 잘 동작하던 프롬프트가 내일은 다르게 동작할 수도 있다는 거예요. 둘째로 추론 과정인 think 태그 내부의 토큰도 과금되기 때문에, 복잡한 작업에서는 예상보다 비용이 높게 나올 수 있어요.

그래서 Joe가 권장하는 전략은 하이브리드 방식이에요. 테스트 작성, 리팩토링, 코드 리뷰 같은 일상적인 코딩 작업에는 MiniMax M2.7을 쓰고, 아키텍처 결정이나 까다로운 버그 디버깅 같은 핵심 작업에만 Claude를 쓰는 거예요. 전체 작업의 70에서 80퍼센트를 저비용 모델로 처리하면 전체 비용을 40에서 60퍼센트 절감할 수 있다고 해요.

이번 경험이 던지는 더 큰 메시지도 있어요. Claude Code의 진짜 가치는 사실 Anthropic 모델에 있는 게 아닐 수 있어요. 어떤 모델과도 연동 가능한 에이전트 인프라를 제공한다는 점, 그리고 그 위에 쌓아온 커스텀 워크플로우가 모델을 바꿔도 그대로 유지된다는 점이 진짜 강점이에요.

그리고 오픈소스 모델과 상용 모델 사이의 격차가 빠르게 좁혀지고 있어요. MiniMax M2 세대를 기준으로 최상위 오픈소스 모델과 GPT-5 같은 상용 모델의 벤치마크 점수 격차가 18점에서 7점으로 줄었다고 해요. 이 추세는 계속 가속화될 가능성이 높아요.

오늘 이야기를 정리하면, MiniMax M2.7은 Claude Code와 조합했을 때 일상적인 코딩 작업에 충분한 성능을 내면서 비용은 크게 줄일 수 있는 실용적인 선택지예요. 모든 작업에 최고급 모델을 쓰는 것보다, 작업의 성격에 맞게 모델을 골라 쓰는 전략이 앞으로 AI 코딩 도구를 효율적으로 활용하는 핵심이 될 것 같아요. 오늘 내용이 도움이 됐으면 좋겠어요. 감사해요.
