맨 위가 Managed 또는 Org 레벨입니다. macOS에서는 `/Library/Application Support/ClaudeCode/CLAUDE.md` 경로에 위치하고, IT 팀이 배포합니다. 핵심은 이겁니다 — 하위 레벨에서 절대 재정의할 수 없습니다. 조직 전체에 적용되는 보안 정책 같은 걸 여기에 넣는 거죠.

로딩 순서는 위에서 아래로. Managed, Project, User, Local 순서입니다. 저자의 표현을 빌리면 "resolution chain", 해석 체인이라고 하는데요, DNS가 도메인을 순차적으로 해석하듯이 설정 충돌이 있으면 상위가 이긴다는 개념입니다.

두 번째 핵심 주제. MCP 설정 스코핑입니다. MCP 서버 설정도 비슷한 계층을 따릅니다. 프로젝트 루트의 `.mcp.json`은 팀 공유용이고 버전 관리에 포함됩니다. 홈 디렉토리의 `~/.claude.json`은 개인용이고 버전 관리에 포함되지 않습니다. 여기서 가장 흔한 안티패턴 두 가지가 나옵니다. 하나는 크레덴셜을 포함해서 전부 프로젝트 파일에 넣는 것 — 비밀값이 버전 관리에 노출됩니다. 다른 하나는 전부 사용자 레벨에 넣는 것 — 팀원이 공유 도구에 접근할 수 없게 됩니다. 올바른 패턴은 서버 정의는 `.mcp.json`에, 크레덴셜은 `~/.claude.json`이나 환경변수에 분리하는 것입니다.

세 번째 핵심. 에이전트당 도구 수는 네다섯 개가 모범 사례입니다. 열여덟 개 도구를 가진 에이전트가 자꾸 엉뚱한 도구를 선택한다면, 해답은 더 나은 프롬프트가 아닙니다. 전문화된 서브에이전트로 분할하는 것이 정답입니다. 그리고 여기서 정말 중요한 포인트 하나. Claude는 도구 이름이 아니라 설명을 보고 도구를 선택합니다. 저자는 이걸 "routing mechanism", 라우팅 메커니즘이라고 표현하는데요, 네트워크에서 패킷이 라우팅 테이블을 보고 경로를 결정하듯이, 도구 설명이 바로 그 라우팅 테이블 역할을 한다는 겁니다. "Analyzes stuff" 같은 모호한 설명은 오라우팅을 유발하고, "Python 소스 파일에 정적 분석을 수행하여 타입 오류, 미사용 import, 스타일 위반을 라인 번호와 함께 반환한다"처럼 구체적으로 써야 정확한 선택이 이루어집니다.

네 번째 핵심. 프로그래밍적 강제 대 프롬프트 기반 안내. 이건 이 시나리오의 황금률이라고 해도 과언이 아닙니다. 저자가 쓴 원문 표현이 인상적인데요, "Prompts execute probabilistically" — 프롬프트는 확률적으로 실행된다. 물리학이나 통계학에서 쓰는 용어를 가져와서, LLM의 본질적 불확실성을 한 단어로 압축한 겁니다. CLAUDE.md에 "커밋 전에 린터를 돌려라"라고 적으면 대부분 따르지만, 컨텍스트가 길어지면 생략될 수 있습니다. 반면 PostToolUse 훅으로 파일 저장 후 자동으로 ESLint가 실행되게 하면, 이건 확정적입니다. 시험에서 "시스템 프롬프트에 지시 추가" 대 "프로그래밍 훅 설정" 중 선택하라면, 훅이 거의 항상 정답입니다.

다섯 번째. Description-Discernment 루프입니다. 팀원마다 Claude의 출력이 다르다는 시나리오가 나오면 주목하세요. CLAUDE.md나 스킬, 도구 설명 같은 Description을 개선하고, 코드리뷰와 테스트를 통한 Discernment로 검증하고, 그 결과를 다시 Description에 반영하는 순환 구조. 저자의 표현으로 "The loop answer is almost always more correct than the one-shot answer" — 루프 답이 일회성 답보다 거의 항상 더 정확합니다. 여기서 one-shot은 머신러닝의 one-shot learning에서 가져온 표현인데, 한 번에 끝내려는 접근법이 아니라 반복적으로 개선하는 접근법이 정답이라는 의미입니다.

마지막으로 안티패턴 정리입니다. 시험에서 오답을 빠르게 제거하는 데 유용합니다. Few-shot 예시로 도구 호출 순서를 제어하려는 것 — 실패합니다. 도구 선택은 설명 기반이지 예시 순서 기반이 아닙니다. 프롬프트로 비즈니스 규칙을 강제하려는 것 — 컨텍스트 압박, 원문에서는 context pressure라고 표현하는데, 컨텍스트 윈도우가 차면서 초기 지시의 우선순위가 낮아지는 현상 때문에 단계가 생략될 수 있습니다. 에이전트에 열다섯 개 이상 도구를 넣는 것 — 중복 설명이 오라우팅을 유발합니다. CLAUDE.md를 두 단계만 알고 있는 것 — managed/org과 local 관련 질문에서 오답을 냅니다.
