# 9달러의 재앙: Anthropic의 하네스 설계 논문이 가르쳐 주는 자율 AI 구축의 교훈

## 1. 요약

- Anthropic의 하네스 설계 논문은 단순 단일 에이전트 접근법의 근본적인 한계를 9달러짜리 실패 사례로 명확히 드러낸다. 단일 Claude 인스턴스로 레트로 게임 제작 도구를 만들려 했을 때, 20분 만에 9달러를 소비하고 작동하지 않는 코드 뭉치를 생산했다.
- 반면 Planner-Generator-Evaluator로 구성된 3중 에이전트 하네스는 6시간, 200달러를 사용해 완전히 기능하는 고품질 애플리케이션을 완성했다. 비용 대비 가치로 보면 "저렴한" 단일 에이전트 접근법이 오히려 무한히 비싸다.
- 논문은 AI 에이전트의 두 가지 핵심 실패 패턴으로 "컨텍스트 불안(context anxiety)"과 "자기평가 편향(self-evaluation bias)"을 지목하며, 이를 극복하기 위한 구조적 분리 원칙을 제시한다.
- GAN(Generative Adversarial Network)에서 영감을 받은 생성자-평가자 분리 아키텍처가 자율 AI 시스템 신뢰성의 핵심이며, 모델 능력이 향상되어도 외부 평가 구조는 유지되어야 한다.
- 하네스의 각 구성 요소는 특정 모델 한계에 대한 반증 가능한 가설을 내포하며, 모델이 발전함에 따라 불필요해진 구성 요소는 제거해야 한다는 진화적 설계 원칙을 강조한다.

## 2. 상세 내용

### 2.1 9달러의 재앙: 단일 에이전트 실험

Claude Opus 4.5 단일 인스턴스에 2D 레트로 게임 제작 도구 구축을 맡겼다. 결과는 다음과 같다.

- **소요 시간**: 20분
- **비용**: 9달러
- **결과물**: 스텁(stub)과 미완성 기능으로 가득 찬 작동 불가능한 프로토타입

에이전트는 코드에 빈 함수와 플레이스홀더 주석이 가득한데도 "작동한다"고 자신 있게 보고했다. 이 현상의 원인은 두 가지 구조적 실패였다.

**컨텍스트 불안(Context Anxiety)**: 컨텍스트 창이 채워질수록 모델은 마치 시험 시간이 끝나가는 학생처럼 서두르기 시작한다. 완전한 구현 대신 플레이스홀더 주석으로 대체하고, 지름길을 택하며, 출력 품질이 체계적으로 저하된다.

**자기평가 편향(Self-Evaluation Bias)**: 모델이 자신이 생성한 코드를 검토할 때, 스텁이 있는 부분을 정신적으로 채워서 읽는다. 실제 코드가 아닌 의도한 기능을 보는 것이다. 이로 인해 오류가 반복 주기 전반에 걸쳐 누적된다.

### 2.2 GAN에서 영감 받은 하네스 아키텍처

Anthropic은 Generative Adversarial Network의 핵심 원리 — 생성자와 비평자의 적대적 긴장 관계 — 를 에이전트 설계에 적용했다. "갈등을 설계함으로써 발전을 설계한다(By engineering conflict, you engineer progress)"는 것이 핵심 통찰이다.

**Planner (계획자)**
- 간단한 브리프를 상세한 제품 명세로 변환
- 격리된 컨텍스트에서 성공 기준과 함께 작업을 이산 단위로 분해
- 의도적으로 고수준 명세를 생성하여 구현 자유도 부여

**Generator (생성자)**
- 깨끗한 컨텍스트로 구현에만 집중
- React, FastAPI, PostgreSQL 등을 사용한 스프린트 방식 기능 구현
- 컨텍스트 오염 없이 Planner의 명세를 충실히 실행

**Evaluator (평가자)**
- Playwright MCP를 사용해 실제 애플리케이션을 실사용자처럼 테스트
- 정적 코드 분석으로는 잡을 수 없는 런타임 버그와 UX 실패를 탐지
- 라우트 순서 오류, 엔티티 연결 누락, 통합 실패 등 실제 오류 발견
- 설계 품질, 독창성, 장인정신, 기능성 4개 차원에서 평가, 생성 주기당 5~15회 실행

### 2.3 완전한 하네스의 결과물

동일한 2D 레트로 게임 제작 도구 과제에 대한 비교:

| 항목 | 단일 에이전트 | 풀 하네스 |
|------|------------|--------|
| 소요 시간 | 20분 | 6시간 |
| 비용 | $9 | $200 |
| 결과물 품질 | 작동 불가, 스텁 가득 | 완전 기능, 완성도 높음 |
| 실질 가치 | 0 (무용지물) | 실사용 가능한 애플리케이션 |

선택은 "9달러의 저렴함" vs "200달러의 고비용"이 아니다. 제로 가치의 망가진 출력물 vs 납품 가능한 소프트웨어 사이의 선택이다.

### 2.4 컨텍스트 관리 전략

단순한 컨텍스트 압축보다 주기적 컨텍스트 리셋이 더 효과적임이 입증됐다. 핵심 전략은 다음과 같다.

- **파일 기반 에이전트 통신**: 구현을 과도하게 제약하지 않으면서 정보 전달 충실도 유지
- **구조화된 핸드오프**: 컨텍스트 리셋 시 이전 상태를 명확하게 전달
- **능동적 컨텍스트 모니터링**: 압축 전략과 리셋 전략의 적절한 조합

### 2.5 진화적 설계: Opus 4.5에서 4.6으로

하네스의 각 구성 요소는 특정 모델 한계에 대한 반증 가능한 가설이다. 모델이 발전하면 구성 요소도 진화해야 한다.

- **제거됨**: 스프린트 경계 (100만 토큰 컨텍스트 창으로 장문맥 한계 해결)
- **진화됨**: 계획이 경직된 방식에서 유연한 방식으로 변화
- **유지됨**: 외부 평가 (자기평가 편향은 아직 해결되지 않은 문제)

비계(scaffolding)는 모델 능력이 향상될수록 줄어들어야 하며, 불필요해진 구성 요소는 제거하고 구성 요소를 비활성화할 수 있도록 설계해야 한다.

## 3. 핵심 인사이트 정리

1. **"저렴한" 실패가 오히려 더 비싸다**: 9달러짜리 단일 에이전트 접근법은 제로 가치 출력물을 생산했다. 실질적으로 무한한 비용이다. 비용 효율성은 절대 비용이 아닌 가치 대비 비용으로 측정해야 한다.

2. **생성자가 자신의 작업을 평가해서는 안 된다**: AI 모델은 자신이 생성한 코드를 검토할 때 병적인 낙관주의자다. 자기평가는 근본적으로 신뢰할 수 없으며, 생성과 평가의 구조적 분리가 필수다. 이 원칙은 모델 성능이 향상되어도 변하지 않는다.

3. **컨텍스트 창은 AI 에이전트의 인지 자원이다**: 컨텍스트가 가득 찰수록 모델의 출력 품질이 저하된다. 컨텍스트를 전략 자원으로 취급하고 능동적으로 관리하는 것이 장기 실행 에이전트의 핵심 설계 원칙이다.

4. **Playwright 기반 라이브 테스트가 정적 분석보다 우월하다**: 평가자가 실제 애플리케이션을 실사용자처럼 테스트하는 것이 코드 리뷰나 정적 분석으로는 잡을 수 없는 런타임 오류와 통합 실패를 발견한다. 이것이 하네스에서 가장 높은 레버리지를 가진 단일 개입이다.

5. **하네스 구성 요소는 가설이지 영구적 구조가 아니다**: 각 구성 요소는 특정 모델 한계를 전제로 존재한다. 그 한계가 해결되면 구성 요소를 제거해야 한다. 불필요한 비계를 쌓아가는 것은 기술적 부채를 늘리는 것과 같다.

6. **평가 기준이 AI의 품질 감각을 형성한다**: 설계 품질, 독창성, 장인정신, 기능성처럼 다차원적 평가 기준을 명시적으로 정의하면 AI의 미적 판단과 품질 방향을 조향할 수 있다. 품질 기준은 코드가 아닌 명세에 있다.

7. **10~25배 비용이 구조적으로 정당화된다**: 하네스 신뢰성 비용은 단일 시도 비용의 10~25배에 달하지만, 이는 반복적 평가와 수정 주기의 필연적 결과다. 이 비용 증가를 숨기지 말고 명시적 트레이드오프로 설계 단계에서 예산에 반영해야 한다.
