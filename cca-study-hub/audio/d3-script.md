CCA 시험 대비, 도메인 쓰리 클로드 코드 설정과 워크플로우입니다. 이 도메인은 전체 시험의 이십 퍼센트, 약 열두 문항을 차지합니다. 단순히 클로드 코드를 어떻게 쓰는지가 아니라, 왜 그렇게 설정해야 하는지, 잘못 설정하면 어떤 장애가 발생하는지를 묻습니다. 핵심 원칙을 먼저 기억하세요. A prompt is guidance, a flag is a guarantee. 프롬프트는 안내이고, 플래그는 보장입니다.

먼저 CLAUDE.md 계층 구조부터 봅니다. 대부분의 학습 자료는 두 개 레벨만 다루지만 공식 문서는 네 개 레벨을 정의하고, 시험은 네 개 모두를 테스트합니다. 최상위는 Managed 또는 Org 레벨입니다. macOS에서는 Library Application Support ClaudeCode 경로에, 리눅스에서는 etc claude-code 경로에 위치합니다. IT나 DevOps 팀이 배포하며 하위 레벨에서 절대 재정의할 수 없습니다. 두 번째는 Project 레벨로, 프로젝트 루트의 CLAUDE.md 또는 .claude 디렉토리 안에 위치합니다. Git으로 공유되므로 팀 전체 코딩 표준에 적합합니다. 세 번째는 User 레벨로, 홈 디렉토리 아래 .claude 폴더에 위치하며 개인 설정이 모든 프로젝트에 적용됩니다. 마지막은 Local 레벨로, CLAUDE.local.md 파일이며 gitignore 대상이라 개인 실험 설정에 씁니다. 해석 순서는 Managed에서 Local 방향이고 상위가 항상 우선합니다. 시험에서 조직 전체 보안 가이드라인이면 Managed Org, 팀 코딩 표준이면 Project, 개인 에디터 선호도면 User, 개인 실험이면 Local이 정답입니다. 모노레포에서는 루트와 하위 디렉토리에 각각 CLAUDE.md를 두면 자동 로드됩니다.

다음으로 CI/CD 필수 플래그를 설명합니다. 파이프라인에서 클로드 코드를 실행할 때 반드시 알아야 할 플래그가 있습니다. 첫째, 대시 p 또는 대시대시 print 플래그입니다. 비인터랙티브 모드로 프롬프트를 받고 처리하고 출력하고 종료하는 원샷 실행입니다. 이걸 빼면 파이프라인이 무한 대기 상태에 빠집니다. 에러가 나는 게 아니라 그냥 멈춥니다. 시험에서 타임아웃을 백이십 분으로 늘리라는 선택지가 나오면 그건 증상 치료이고, 대시 p 추가가 근본 원인 해결입니다. 둘째, 대시대시 bare 플래그입니다. 훅, LSP, 플러그인 동기화, 스킬 디렉토리 스캔, 자동 메모리, OAuth 인증을 전부 스킵합니다. 대시 p만 쓰면 로컬 머신과 CI 러너에서 다른 동작이 발생할 수 있는데 bare가 이 차이를 제거해서 재현성을 보장합니다. 중요한 건 bare가 OAuth를 스킵하므로 반드시 ANTHROPIC_API_KEY 환경변수를 명시적으로 설정해야 합니다. Anthropic은 향후 bare를 대시 p의 기본값으로 만들 예정입니다. 셋째, 대시대시 output-format json입니다. 클로드의 응답을 항상 유효한 JSON으로 출력합니다. 프롬프트에 Always return JSON이라고 쓰면 구십 퍼센트만 작동합니다. 마크다운으로 감싸거나 설명 텍스트가 붙을 수 있거든요. 플래그는 백 퍼센트 보장입니다. 대시대시 json-schema와 조합하면 특정 스키마로 출력을 제약할 수 있는데, 이때 결과는 structured_output 필드에 위치합니다. result가 아닙니다. 이건 단순 암기 문제로 자주 출제됩니다.

이제 가장 중요한 구분인 대시대시 tools와 대시대시 allowedTools의 차이입니다. tools에 Read와 Bash를 지정하면 클로드는 그 두 개만 사용할 수 있고 나머지는 전부 차단됩니다. 이것이 실제 샌드박싱입니다. 반면 allowedTools에 같은 도구를 지정하면 그 도구들의 권한 프롬프트만 스킵될 뿐 다른 도구도 여전히 사용 가능합니다. 편의 기능이지 보안이 아닙니다. 시험에서 CI/CD에서 의도하지 않은 동작을 막으려면이라고 물으면 tools가 정답이고 allowedTools는 대표적인 함정 오답입니다.

CI/CD 파이프라인 통합 패턴을 봅니다. 개발자가 대기하는 실시간 작업이면 Real-Time API를 씁니다. 야간이나 주간 배치 작업이면 ZDR이 필요한지 확인합니다. ZDR이 필요하면 Real-Time API에 Prompt Caching을 더하고, 불필요하면 Batch API로 오십 퍼센트 비용을 절감합니다. 자동 PR 코드 리뷰가 가장 빈출 패턴인데, bare 대시 p output-format json tools 네 가지를 모두 포함해야 합니다. 재시도 루프에서는 이삼 회 제한이 철칙이고, 이전 에러를 다음 재시도에 반드시 포함해야 합니다. 에러 컨텍스트 없이 다시 시도하는 blind retry는 안티패턴입니다.

Custom Skills와 Rules의 차이입니다. Rules는 .claude rules 디렉토리에 위치하고 매 세션 자동 로딩되어 항상 컨텍스트에 포함됩니다. Skills는 .claude skills에 위치하고 호출 시에만 로딩됩니다. 삼 단계 이상의 워크플로우가 두 번 이상 반복되면 스킬로 인코딩합니다. 스킬은 CLAUDE.md를 참조해야지 복제하면 안 됩니다. 이중 유지보수를 피하기 위해서입니다.

Hooks 시스템입니다. CLAUDE.md에 항상 린터를 실행하라고 써도 컨텍스트 윈도우가 채워지는 컨텍스트 압력 하에서 단계가 생략될 수 있습니다. 프롬프트는 확률적이고 훅은 결정적입니다. PreToolUse는 도구 사용 전에 실행되어 승인된 명령어 목록을 확인합니다. PostToolUse는 도구 사용 후에 실행되어 파일 저장 후 자동 린트 같은 작업을 합니다. 시험에서 시스템 프롬프트에 지시를 추가하는 것과 프로그래밍적 훅 설정 중 선택하라면 훅이 거의 항상 정답입니다.

ZDR과 Batch API입니다. Real-Time API는 즉시 응답하고 ZDR과 호환됩니다. Batch API는 최대 이십사 시간 처리 윈도우를 가지며 오십 퍼센트 할인이지만 ZDR 대상이 아닙니다. 이것이 가장 중요한 시험 팩트입니다. 규제 산업에서 ZDR이 필요하면 Batch API는 절대 사용할 수 없습니다. 시험에서 규제 산업과 Batch API가 같은 선택지에 있으면 즉시 제거하세요.

Task 도구는 서브에이전트를 독립된 컨텍스트에서 생성합니다. 각 서브에이전트는 독립적으로 작업하고 결과만 부모에게 반환합니다. 도메인 원의 컨텍스트 격리 원칙의 실제 구현입니다. 컨텍스트를 공유한다는 선택지는 오답입니다.

continue와 resume 플래그입니다. continue는 마지막 대화를 이어가는 것이고 resume은 특정 세션 ID를 지정해서 재개하는 것입니다. CI/CD 파이프라인에서 멀티턴이 필요하면 resume을 써야 합니다. 병렬 파이프라인에서 어떤 세션이 마지막인지 보장할 수 없기 때문입니다.

Memory 시스템은 대화 중 중요 사실을 자동 저장하는데, bare 모드에서 비활성화됩니다. 이유는 CI/CD 재현성입니다. 인증은 인터랙티브 모드에서 OAuth, bare 모드에서 API Key를 사용합니다.

핵심 안티패턴 다섯 가지를 정리합니다. 첫째, CI/CD에서 대시 p 없이 실행하면 무한 대기. 둘째, 프롬프트로 JSON 요청하면 구십 퍼센트만 보장. 셋째, allowedTools로 도구를 제한하려는 시도는 보안이 아님. 넷째, 팀 표준을 User 레벨에 두면 Git 비공유로 확장 불가. 다섯째, 규제 산업에서 Batch API 사용은 컴플라이언스 위반.

마지막으로 암기 숫자입니다. Prompt Caching 읽기 절감률 구십 퍼센트, 쓰기 추가 비용 이십오에서 백 퍼센트. Batch API 절감 오십 퍼센트, 처리 윈도우 이십사 시간. 재시도 제한 이삼 회. 에이전트당 권장 도구 수 사오 개. CLAUDE.md 레벨 네 개. 선택지에 나오면 즉시 판별할 수 있어야 합니다. 마지막으로 핵심 한 줄입니다. CI/CD에서 대시 p 대시대시 bare는 선택이 아니라 필수이고, tools는 제한이며 allowedTools는 편의일 뿐입니다.
