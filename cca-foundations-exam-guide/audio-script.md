# CCA 딥다이브 시리즈 에피소드 1 — CCA Foundations 시험 전체 지형

안녕하세요, CCA 딥다이브 시리즈 첫 번째 에피소드입니다. 오늘은 CCA Foundations 시험의 전체 지형을 살펴보겠습니다.

오늘 다룰 내용은 Rick Hightower라는 분이 쓴 CCA Foundations 시험 가이드입니다. 이 분은 Fortune 100 금융사에서 ML과 AI 개발을 이끈 기술 임원이고요, 8부작 시리즈의 첫 번째 글로 시험의 전체 그림을 그려줍니다.

# 왜 지금 CCA인가

자, 먼저 이 자격증이 왜 중요한지부터 이야기해볼게요. 2026년 3월 12일에 Anthropic이 Claude Certified Architect, 줄여서 CCA Foundations 시험을 출시했습니다. 여기서 중요한 건요, 이건 프롬프트를 잘 쓰는지 보는 시험이 아닙니다. 실제로 Claude를 가지고 프로덕션 시스템을 설계할 수 있는가, 이걸 검증하는 AI 업계 최초의 전문 자격증이에요.

그런데 자격증 자체보다 더 주목해야 할 게 있습니다. Anthropic이 이 시험과 함께 1억 달러 규모의 Claude Partner Network 투자를 발표했거든요. 어떤 파트너들이 참여하냐면요, Accenture가 3만 명 전문가 교육을 시작했고, Cognizant은 35만 명에게 Claude 접근권을 부여했고, Deloitte는 무려 47만 명을 플랫폼에 투입했습니다. 이건 파일럿이 아닙니다. 수십만 명 단위의 인력 전환이에요.

원문에서 저자가 이런 표현을 썼는데요, "Let that number land for a moment"라고 했습니다. 숫자가 착지하도록 잠시 기다리라는 건데, 비행기가 착륙하듯 정보가 뇌에 착륙하는 순간을 말하는 거예요. 1억 달러라는 숫자의 무게를 직접 느껴보라는 뜻이죠. 이 모든 조직에 Claude 기반 시스템을 아키텍팅할 사람이 필요하고, CCA가 바로 그 최초의 표준화된 검증 신호입니다.

# 시험 형식

시험 형식을 먼저 짚어드릴게요. 60문항, 120분, 문항당 약 2분입니다. 합격 점수는 1000점 만점에 720점이고, 비용은 99달러예요. 꼭 기억하세요, 이건 301 레벨 시험입니다. 미국 대학 과목 번호 체계에서 300번대는 고급 전공 수준이에요. 최소 6개월 이상 실무 경험이 있는 사람을 대상으로 합니다.

형식은 시나리오 기반 객관식인데, 150에서 200단어 정도의 시나리오를 읽고 최적의 아키텍처 결정을 선택하는 방식입니다. 감독관이 모니터링하고, 일시 중지도 안 되고, 외부 도구도 못 씁니다.

원문에서 저자가 "the knowledge has to live in your head, not in a browser tab"이라고 했는데요, 개발자들이 수십 개 탭을 열어놓고 작업하는 습관을 정확히 겨냥한 표현이에요. 탭에 사는 지식이 아니라 머릿속에 상주하는 지식이어야 한다는 거죠.

시간 관리 팁도 하나 드리면, 초기 응시자 피드백에 따르면 1차로 확실한 문제를 먼저 쭉 풀고, 망설여지는 건 플래그를 걸어두고 남은 시간에 돌아오는 게 좋다고 합니다. 한 문제에 5분 이상 쓰지 마세요.

# 5개 역량 도메인

자, 이제 핵심입니다. 시험은 5개 역량 도메인으로 나뉘어요.

첫 번째, Agentic Architecture. 비중이 27%로 가장 큽니다. 멀티 에이전트 설계와 coordinator에서 subagent로 위임하는 패턴이 핵심인데요, 여기서 중요한 건요, 서브에이전트는 컨텍스트를 자동으로 상속하지 않습니다. 빈 상태에서 시작해요. 명시적으로 전달해줘야 합니다. 이게 시험에서 가장 많이 나오는 개념이에요. 그리고 15개 이상 도구를 가진 단일 에이전트, 이른바 슈퍼 에이전트 패턴이 나오면 거의 항상 오답입니다. 에이전트당 4개에서 5개 도구가 적정선이에요.

에스컬레이션 로직도 중요합니다. Claude가 스스로 "자신감이 없다"고 판단해서 에스컬레이션하는 방식은 오답이에요. 결정론적 규칙, 예를 들어 거래 금액이 1만 달러 초과라거나 계정 등급이 enterprise라거나, 이런 명확한 기준으로 판단해야 합니다.

두 번째, Claude Code. 20% 비중이에요. CLAUDE.md 파일의 계층 구조를 알아야 합니다. 프로젝트 레벨과 사용자 레벨이 있고, 개인 설정을 프로젝트 파일에 넣는 건 안티패턴입니다. 그리고 CI/CD에서는 반드시 마이너스 p 플래그를 써서 비대화 모드로 실행해야 해요. 이거 빠뜨리면 시스템이 멈춥니다. 거기에 대시대시 bare 플래그로 재현성을 보장하고, 대시대시 output-format json으로 파이프라인에서 파싱할 수 있게 출력합니다.

계획 모드와 직접 실행의 구분도 알아야 해요. 복잡한 멀티 파일 변경이면 계획 모드, 명확한 저위험 단일 작업이면 직접 실행. 시나리오에서 복잡한 변경이 나오는데 직접 실행을 선택하면 안티패턴입니다.

세 번째, Prompt Engineering. 역시 20%입니다. 꼭 기억하세요, 프롬프트로만 JSON 형식을 강제하려는 건 거의 항상 오답이에요. "시스템 프롬프트에 지시를 추가한다"거나 "더 상세한 포맷팅 가이드를 넣는다" 같은 선택지가 보이면 의심하세요. 원문에서 저자가 "Prompts are guidance. Code is law."라고 했는데, 이게 이 도메인의 핵심입니다. 프롬프트는 안내이고 코드가 법이에요. 정답은 tool_choice와 입력 스키마, structured outputs API, 그리고 validation에서 retry로 이어지는 루프입니다.

Stop reason도 반드시 외워야 합니다. tool_use는 도구 결과를 기다리는 중이고, end_turn은 응답 완료, max_tokens는 토큰 한도 도달이에요. tool_use stop reason을 확인하지 않으면 도구 결과를 완전히 놓치게 됩니다.

네 번째, Tool Design과 MCP. 비중은 18%로 낮은 편인데, 여기서 중요한 건요, 이 도메인이 다크호스입니다. 초기 응시자들이 가장 많은 예상 외 실점을 보고한 영역이에요. MCP에는 세 가지 프리미티브가 있습니다. Tool은 실행 가능한 함수, Resource는 읽기 전용 데이터, Prompt는 재사용 템플릿. Claude가 어떤 행동을 취해야 하면 Tool이고, 컨텍스트로 읽기만 하면 Resource입니다. 이 경계를 정확히 구분하는 게 핵심이에요.

그리고 Tool Description, 즉 도구 설명이 라우팅을 결정한다는 것도 꼭 기억하세요. Claude는 에이전트 이름이나 도구 이름이 아니라 설명을 보고 어떤 도구를 호출할지 결정합니다. "handles customer requests" 같은 모호한 설명은 잘못된 도구 선택으로 이어져요.

에이전트당 도구는 4개에서 5개가 적정선입니다. 18개 도구를 가진 에이전트는 선택 정확도가 떨어집니다. 초과분은 전문 서브에이전트로 분산하세요.

다섯 번째, Context Management. 15%로 비중은 가장 낮지만 다른 모든 도메인에 영향을 미칩니다. Lost in the Middle이라는 효과가 있는데요, 컨텍스트 창의 중간에 묻힌 정보는 모델이 주의를 덜 기울인다는 겁니다. 이건 컨텍스트 창 크기와 무관해요. 200K 토큰 창에서도 중간의 주의력 저하는 발생합니다. 중요한 정보는 처음이나 끝에 배치하세요.

API별 비용 최적화도 나옵니다. 세 가지를 비교해야 해요. Prompt Caching은 최대 90% 절감에 실시간 지연, 반복 시스템 프롬프트에 적합합니다. Batch API는 50% 절감이지만 최대 24시간 걸려요, 야간 감사나 대량 처리용입니다. Real-Time API는 표준 가격에 실시간 지연, 사용자 대면 워크플로우용이고요. 사용자가 응답을 기다리는 실시간 상황에서 비용을 줄이려면 답은 절대 Batch API가 아니라 Prompt Caching입니다. Batch는 최대 24시간 걸리니까요.

# 6개 프로덕션 시나리오

시험에는 6개 프로덕션 시나리오가 있고, 시험마다 그중 4개가 랜덤으로 출제됩니다. 그러니까 전부 준비해야 해요.

간단히 훑어볼게요. 고객 지원 에이전트 시나리오에서는 에스컬레이션을 Claude의 자체 신뢰도 점수로 결정하면 오답입니다. 결정론적 규칙, 그러니까 금액이나 계정 등급 같은 명확한 기준으로 판단해야 합니다. 코드 생성 시나리오에서는 큰 컨텍스트 창이 주의력 분산을 해결한다고 생각하면 틀려요. Lost in the Middle은 창 크기와 무관합니다. 멀티 에이전트 리서치에서는 아까 말씀드린 슈퍼 에이전트 함정, 그리고 서브에이전트 컨텍스트 비상속이 나옵니다.

개발자 생산성 도구 시나리오에서는 Plan Mode와 Direct Execution 판단, CLAUDE.md 계층 설정이 핵심이에요. CI/CD 시나리오에서는 마이너스 p 플래그가 핵심이고요, 대시대시 bare와 대시대시 output-format json도 함께 씁니다. 구조화 데이터 추출에서는 프롬프트만으로 JSON을 강제하면 오답입니다. tool_choice와 스키마, 검증-재시도 루프가 정답이에요.

# 합격의 5가지 멘탈 모델

이제 합격자를 만드는 5가지 멘탈 모델을 정리해드릴게요. 이건 시험 전체를 관통하는 원칙들입니다.

첫째, 프롬프트는 가이드이고 코드는 법이다. 원문에서 저자가 "Prompts are guidance. Code is law."라고 썼는데요, 이건 블록체인의 Code is law 철학에서 차용한 표현이에요. 신뢰성이 필요하면 답은 항상 코드입니다. 시스템 프롬프트에 지시를 추가한다는 선택지가 보이면 거의 항상 오답이에요.

둘째, 서브에이전트는 컨텍스트를 상속하지 않는다. 빈 슬레이트에서 시작하니까 명시적으로 전달해야 합니다. 이게 시험에서 가장 많이 나오는 단일 개념이에요.

셋째, Tool Description이 라우팅을 결정한다. 에이전트 이름이 아니라 도구의 설명을 보고 선택하기 때문에, 설명이 모호하면 잘못된 도구가 호출됩니다.

넷째, Lost in the Middle 효과는 실재한다. 긴 컨텍스트를 다루는 모든 시나리오에 적용하세요. 중요 정보는 처음이나 끝에 배치합니다.

다섯째, API를 지연 시간 요구에 맞춰라. 사용자가 기다리면 실시간 API에 Prompt Caching, 배경 작업이면 Batch API입니다.

# 시험 전략과 주의사항

초기 응시자들의 피드백을 종합하면 몇 가지 중요한 점이 있습니다. 먼저, 오답지가 매우 그럴듯하게 설계되어 있습니다. 원문에서 distractor라는 시험학 전문 용어를 쓰는데, 정답이 아닌 선택지가 방해물처럼 기능한다는 뜻이에요. 실전 경험이 없으면 정답과 그럴듯한 오답을 구분하기 어렵습니다.

그래서 안티패턴을 먼저 인식하는 전략이 효과적입니다. 4개 선택지를 보면, 먼저 안티패턴을 제거하세요. 슈퍼 에이전트가 보이면 제거, 프롬프트로만 강제가 보이면 제거, 모델의 자체 신뢰도 점수가 보이면 제거. 남은 선택지에서 정답을 고르는 게 훨씬 쉽습니다.

시나리오 체인도 주의해야 합니다. 한 문제의 컨텍스트가 다음 문제에 이어지는 연속 문제가 있어서, 앞 문제를 틀리면 뒤 문제도 흔들릴 수 있어요. 시간 관리가 중요한 이유입니다.

Reddit에서 985점 보고도 있으니 만점에 가까운 점수가 가능하고, 합격선 720은 의미 있는 오차 범위를 제공합니다.

# 4주 학습 계획

저자가 제안하는 학습 계획은 4주짜리입니다. 총 30시간에서 37시간 정도 걸려요.

1주차에는 Claude 101과 AI Fluency Framework로 기본 어휘와 멘탈 모델을 내재화합니다. agentic loop, context forking, stop reason, tool_choice 같은 용어들이요.

2주차가 가장 중요합니다. Building with the Claude API 과정에 8시간에서 10시간을 투자하세요. validation에서 retry로 이어지는 루프를 직접 만들어보고, tool_choice로 구조화 출력을 다뤄보고, 도구 3개에서 4개를 가진 에이전트를 구축해봐야 합니다.

3주차에는 MCP 서버를 직접 구축합니다. Tool 3개와 Resource 1개로 구성된 서버를 만들어보세요. 그리고 CLAUDE.md 설정과 CI/CD에서 마이너스 p 플래그로 실행하는 것도 다룹니다.

4주차에는 모의 시험입니다. 실전 조건, 그러니까 노트 없이 문서 없이 풀어보고, 900점 이상 나오면 본 시험에 등록하세요.

원문에서 저자가 "If you have not built an MCP server by the time you take the exam, you are gambling on this domain"이라고 했어요. 도박이라는 단어를 쓴 건, MCP Tool과 Resource의 경계는 실제로 만들어봐야만 체화된다는 걸 강조하기 위해서입니다. 저자의 어투가 부트캠프 수석 강사 같다고 하는 이유가 이런 데 있어요. "Do not skip the exercises." 이렇게 부드러운 권유가 아니라 명령이 먼저 옵니다.

# 암기해야 할 핵심 수치

시험 전에 꼭 외워둬야 할 수치들을 한 번 더 정리합니다. 시험은 60문항 120분, 합격은 720점, 비용은 99달러. 에이전트당 적정 도구 수는 4개에서 5개, 15개 이상이면 슈퍼 에이전트 안티패턴입니다. Prompt Caching은 최대 90% 절감, Batch API는 50% 절감이지만 최대 24시간 지연, 프로덕션 시나리오는 6개 중 4개가 랜덤 출제됩니다. 모의 시험에서 900점 이상 나온 후에 본 시험을 등록하라고 저자가 권합니다.

# 마무리

정리하겠습니다. CCA Foundations는 AI 업계 최초의 프로덕션 아키텍처 자격증이고, 99달러에 시장 선점이 가능한 기회입니다. 원문 저자가 "You showing up with a CCA badge while the market is still forming is a different proposition"이라고 했는데요, 시장이 형성되는 지금 배지를 들고 나타나는 것과 2년 뒤 모두가 가지고 있을 때 나타나는 건 전혀 다른 포지셔닝이라는 뜻이죠.

시험의 핵심은 무엇을 해야 하는가만큼 무엇을 하면 안 되는가를 아는 것입니다. 안티패턴을 인식하는 속도가 합격을 결정합니다. 그리고 준비 과정 자체가 실력 향상이에요. 시험을 보지 않더라도 4주 학습 계획을 따르면 tool design, context management, agentic architecture의 엣지 케이스를 발견하게 됩니다.
